## プロジェクトのウェイトリストフロー概要

## 1. 初期設定（管理者 — A1）(A1-004)

Link figma: https://www.figma.com/design/hK2f5YpFcnfzNS6KJTMP40/AIOrder?node-id=4529-21863&t=9YgqsCUQOe0CHjsi-0

運用前に、管理者が 機能別詳細設定 → 待ち順番・受付設定 で以下を設定します：

- ウェイトリスト受付のON/OFF：受付を開始するときは「Waitlist利用」、一時停止するときは「休止」に切り替える（例：満席時や営業時間外）。
- 受付チャンネル：オンライン（Web / LINE）経由、またはカウンターのみ（発券機によるオフライン受付）を選択。
- 受付日時：月〜金、土曜日、日曜日それぞれ個別に設定。祝日の受付を追加することも可能。
- 顧客へのリマインド：「何グループ前になったらLINE通知を送るか」を設定（例：残り3グループ）、通知頻度（5分ごと）、最大通知回数。
- 自動キャンセル：呼び出し後10分間応答がない場合、自動的にキャンセルする。
- オフライン発券機：USB接続のプリンターがある場合、スマートフォンを持っていないウォークイン顧客に対して物理的な番号札（A001、A002...形式）を発行できる。

## 2. ウェイトリスト登録方法（3通り）

### 方法1 — LINE Mini App からの自己登録（C1）

Link Figma: https://www.figma.com/design/hK2f5YpFcnfzNS6KJTMP40/AIOrder?node-id=4529-21317&t=9YgqsCUQOe0CHjsi-0

来店前または移動中の顧客向けのメインフローです：

1. 顧客がレストランのLINE Mini Appを開き、「オンライン順番待ち」を選択する。(C1-001)
2. 現在の状況が表示される：待機グループ数、待ち時間の目安（分）。(C1-007)
3. 「順番を取る」をタップ → フォームに入力する：(C1-008)
- 人数（必須）
- 氏名（必須）
- 電話番号（必須）
- メールアドレス（任意）
- 来店予定時間（任意）
- 希望席タイプ（テーブル席 / 個室 / カウンター席...）
4. 登録後、例えば「R-0005」のような番号と待ち時間の目安（約25分）、チェックイン用QRコードを受け取る。(C1-009)
5. アプリ内でリアルタイムに待ち状況を確認できる：現在呼ばれている番号、自分の順番、残り待ち時間。(C1-010)
6. 順番が近づくと、LINEが自動的にリマインド通知を送信する（残りXグループ）。

### 方法2 — カウンターでの登録（S1）

1. 顧客がフロントに設置されたタブレットS1を使用し、ウェイトリスト登録ボタンをタップする。(1.1.1)
2. 顧客がフォームに入力する：(1.1.2)
- 人数（必須）
- 氏名（必須）
- 電話番号（必須）
- メールアドレス（任意）
- 来店予定時間（任意）
- 希望席タイプ（テーブル席 / 個室 / カウンター席...）
3. 登録後、例えば「R-0005」のような番号と待ち時間の目安（約25分）を受け取り、チェックインに使用する。(1.2.3)
4. 順番が近づくと、LINEが自動的にリマインド通知を送信する（残りXグループ）。

### 方法3 — S2 スタッフが顧客の代わりに登録する

アプリを使いたくないウォークイン顧客向け：

1. フロントスタッフがRestaurant Board S2 のウェイトリスト管理画面で「新規来店登録」をタップし、情報入力ポップアップを開く。
2. 電話番号で既存顧客を検索 — 常連の場合は氏名が自動入力される。
3. 人数、希望席タイプ、来店チャンネル（ウォークイン / 電話...）を入力する。
4. 人数、希望席タイプ、現在のテーブル状況（空席 / 片付け中）に基づいて適切なテーブルをシステムが提案する。
5. スタッフがテーブルを選択 → 確認 → システムが番号を発行する。
6. 物理的な発券機がある場合、顧客用の番号札を自動印刷する。
7. 顧客がLINE連携済みの電話番号を入力した場合、システムがLINE通知を送信する。

## ウェイトリスト管理（スタッフ — S2 Restaurant Board）

顧客の待機中、スタッフはRestaurant Board の画面から管理します：

- リアルタイム概要の確認：使用中テーブル数 / 空きテーブル数 / 待機グループ数。(2.1.1)
- 待ちリストは番号順に表示：氏名、人数、席タイプ、待ち時間、登録チャンネル（LINE / ウォークイン / Web）。
- 番号の呼び出し：スタッフが次の番号を呼び出すと、システムが自動的に：
    - 該当顧客にLINE通知を送信
    - S3 ディスプレイ（ロビーの呼び出し案内板、設置している場合）に呼び出し番号を表示
- X分後に顧客が来なければ → スタッフが再呼び出し またはキャンセル → 次の番号へ移行。
- 顧客をテーブルに案内したら → テーブルの状態を「空席」から「使用中」に更新する。

## ロビー表示画面（S3 — オプション）

Link figma: https://www.figma.com/design/hK2f5YpFcnfzNS6KJTMP40/AIOrder?node-id=4529-21315&t=9YgqsCUQOe0CHjsi-0

店舗にディスプレイを設置している場合：

- 現在呼ばれている番号と次に呼ばれる番号を自動更新。
- 状態表示：「ご案内中」/ 「まもなく呼び出し」/ 「お待ちください」。
- ロビーで待機中の顧客がスマートフォンを見なくても確認できる。

## LINE による自動通知（A1_010）

すべての通知は管理者画面でテンプレートを事前設定します：

---

参加者まとめ

管理者 (A1)  →  ウェイトリストの全ルールを設定
↓

顧客 (C1)  →  LINE Mini App から自己登録（オンライン）
↓

フロント (S1/S2) →  カウンターでの登録サポート + ウェイトリスト管理
↓

ディスプレイ (S3) →  ロビーでの呼び出し番号表示（オプション）
↓

顧客 → チェックイン → テーブルへ案内 → ウェイトリスト終了
